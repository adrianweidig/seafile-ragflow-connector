# Internationalization and Unicode

🌐 Languages: [Deutsch](../i18n.md) | **English**

## Default and Fallback

German is the default language for human-readable connector output, dashboard
text, OpenWebUI artifacts, the main README, community files, and default
documentation. English is the primary alternative language.

Language selection follows this order:

1. explicit user setting, for example `CONNECTOR_LANGUAGE=de` or `CONNECTOR_LANGUAGE=en`
2. project settings
3. system locale when Python can report it reliably
4. locale environment variables such as `LC_ALL`, `LC_MESSAGES`, `LANGUAGE`, or `LANG`
5. stable fallback to German

Unknown or minimal locale values such as `C` or `POSIX` are ignored. Container,
systemd, and headless environments therefore remain reproducible: when no
reliable language is available, German is used.

## Resources

Translations live in:

```text
src/seafile_ragflow_connector/locales/
  de.json
  en.json
```

The central API is `seafile_ragflow_connector.i18n.Localizer`. New keys are
grouped by domain, for example `cli.*`, `sources.*`, `preview.*`, or
`openwebui_artifact.*`. Missing keys do not execute code or trigger dynamic
imports; they fall back to German or to the key text.

## Interfaces

- CLI help and selected error messages use the package resources.
- The dashboard provides a visible language selector, uses browser locale as a
  convenience signal, and stores an explicit choice locally in the browser.
- OpenWebUI artifacts receive a `LANGUAGE` valve so additional language logic
  can be added without changing machine-readable IDs.
- API field names, internal status codes, environment variable names, and
  protocol values are not translated.

## GitHub Structure

GitHub does not automatically translate the regular repository view. The
project therefore uses explicit files and links:

- `README.md` is German.
- `README.en.md` is English.
- `docs/de/index.md` is the German documentation entry.
- `docs/en/index.md` is the English documentation entry.
- Community files without a suffix are German.
- Community files with `.en.md` are English.
- Issue and PR templates contain visible German/English labels.

## UTF-8 and Unicode

All resources and documentation remain UTF-8. Tests cover umlauts, `ß`,
non-Latin characters, and emoji. Translation resource file operations use
explicit UTF-8. The project does not transliterate text automatically; technical
IDs and paths are not rewritten by language logic.

## Adding a Language

1. Add a resource file under `src/seafile_ragflow_connector/locales/<code>.json`.
2. Add `<code>` to `SUPPORTED_LANGUAGES` in `src/seafile_ragflow_connector/i18n.py`.
3. Add tests for locale detection, fallback behavior, and representative output.
4. Extend language links in the README files, docs entries, and relevant community files.
5. For GitHub Pages or static pages, keep a visible manual language switch and
   use German as the safe default route.
